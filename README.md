### 위기상황 시뮬레이션 (UML 설계 팀 프로젝트)

## 1. 프로젝트 개요

본 프로젝트는 위험 지역 내에서 로봇이 스스로 지도를 작성하고, 장애물을 회피하며 목표 지점까지 도달하는 과정을 시뮬레이션하는 C# 기반의 소프트웨어입니다. UML 설계를 바탕으로 구현되었으며, 로봇의 하드웨어(센서, 구동)와 소프트웨어(길찾기, 지도 작성)를 분리하여 설계했습니다.

## 3. 주요 기능 및 알고리즘

### 핵심 기능

- **자율 주행 시뮬레이션**: 로봇이 센서를 통해 주변 환경(장애물, 목표물, 위험지역)을 인식합니다.
- **지도 작성 (Mapping)**: 탐색한 구역의 정보를 바탕으로 내부적으로 `MiniMap`을 갱신합니다.
- **경로 탐색 (Pathfinding)**: 현재 위치에서 목표 지점까지의 최적 경로를 계산합니다.
- **장애물 회피**: 이동 중 장애물 발견 시 경로를 재탐색합니다.

### 기술 스택

- **Language**: C#
- **Algorithm**: A\* (A-Star) Pathfinding Algorithm
- **Design Pattern**: UML 기반의 객체지향 설계

## 4. 시스템 아키텍처

시스템은 크게 세 가지 모듈로 구성되어 있습니다.

1.  **SIM (Simulation Layer)**

    - `CompleteMap`: 실제 환경의 정답 지도(Ground Truth)를 관리합니다.
    - `RobotMovementInterface`: 로봇의 하드웨어를 시뮬레이션하며, 이동 명령을 수행하고 센서 데이터(`SensorReport`)를 반환합니다.

2.  **Navigator (Logic Layer)**

    - `Make_Path`: A\* 알고리즘을 사용하여 최적의 경로를 계산합니다.
    - `Minimap`: 로봇이 센서를 통해 파악한 불완전한 지도 정보를 저장합니다.
    - `MapReport`: 현재 파악된 지도 상태를 상위 모듈로 보고합니다.

3.  **GUI / TestApp (Presentation Layer)**
    - `Interface`: 사용자의 입력을 받아 맵을 설정하고 시뮬레이션을 단계별로 진행(`doNext`)합니다.
    - `MapEditor`: 초기 맵 크기, 장애물 위치, 목표 지점 등을 설정합니다.
